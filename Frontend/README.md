# Трамплин — Карьерная платформа (Фронтенд)

## 📌 О проекте

Цифровая экосистема для взаимодействия студентов, выпускников и IT-работодателей. Проект разрабатывается в рамках олимпиады IT-Планета 2026.

Этот README предназначен для передачи контекста другому фронтенд-разработчику (или нейронке), который будет продолжать работу над проектом. Здесь описана текущая структура, реализованный функционал и то, что предстоит сделать.

## ✅ Реализованный функционал

### Главная страница (`/`)

| Элемент | Описание |
|---------|----------|
| **Яндекс.Карты** | Интерактивная карта с маркерами, умное позиционирование тултипов, разнесение маркеров в одной точке |
| **Маркеры** | Разные цвета для разных типов (вакансия/стажировка/менторство), клик открывает модалку с деталями. **Избранные вакансии выделяются отдельным цветом и звёздочкой** |
| **Дровер (Drawer)** | Выдвижная панель снизу, открывается по клику на полоску или свайпу вверх, закрывается свайпом вниз |
| **Список возможностей** | Карточки с названием, компанией, навыками, зарплатой, датами, кнопкой избранного |
| **Фильтрация** | По навыкам, уровню (Стажёр/Junior/Middle/Senior), типу (Стажировка/Вакансия/Мероприятие/Менторство) |
| **Поиск** | По названию, компании, навыкам, городу — комбинируется с фильтрами. **Поиск синхронизирован с картой** |
| **Избранное** | Сохраняется в localStorage, синхронизация между главной страницей и личным кабинетом |
| **Уведомления** | Модалка с моковыми уведомлениями, счётчик на иконке колокольчика, отметка о прочтении |
| **Профиль (меню)** | Выпадающее меню с кнопками "Профиль" и "Выйти" |

### Авторизация и регистрация (`/login`)

| Элемент | Описание |
|---------|----------|
| **Вход** | Форма с email и паролем, валидация, обработка ошибок |
| **Регистрация** | Два шага: 1) email и пароль, 2) выбор роли и заполнение профиля |
| **Роли** | Соискатель (вуз, ФИО), Работодатель (компания, ИНН, сайт), Куратор (админ) |
| **Восстановление пароля** | Модалка с отправкой инструкции (мок) |

### Личный кабинет соискателя (`/profile/applicant`)

| Элемент | Описание |
|---------|----------|
| **Профиль** | ФИО, вуз, год выпуска, навыки, GitHub, загрузка резюме (PDF, до 5 МБ) |
| **Редактирование профиля** | Модалка с формой, валидация, сохранение в стейте |
| **Приватность** | Три уровня: "Все авторизованные", "Только я", "Только контакты". Сохраняется в localStorage |
| **Отклики** | Список откликов со статусами (Принят/Отклонён/На рассмотрении), удаление отклика |
| **Избранное** | Список сохранённых вакансий, удаление из избранного, синхронизация с главной страницей |
| **Проф. контакты** | Список контактов (друзей), добавление через поиск, удаление, просмотр профиля |
| **Рекомендации** | Входящие рекомендации от контактов |
| **Карьерный трекер** | Мероприятия и собеседования: добавление (модалка с датой и временем), удаление, сворачивание списка |
| **Кнопки "Показать все"** | Для всех разделов — по умолчанию показывается 3 элемента |

### Личный кабинет работодателя (`/profile/employer`)

| Элемент | Статус |
|---------|--------|
| **Интерфейс** | ✅ Свёрстан, адаптирован под тёмную тему |
| **Профиль компании** | Название, ИНН, сфера деятельности, сайт, Telegram, адрес |
| **Управление возможностями** | Создание, редактирование, удаление вакансий/стажировок |
| **Управление откликами** | Просмотр откликов, изменение статуса (Принять/Отклонить/В резерв), просмотр профиля соискателя |
| **Логика** | 🟡 Не реализована, используется статический контент из моков |

### Личный кабинет куратора (`/profile/curator`)

| Элемент | Статус |
|---------|--------|
| **Интерфейс** | ✅ Свёрстан, адаптирован под тёмную тему |
| **Верификация компаний** | Просмотр заявок, подтверждение/отклонение |
| **Модерация карточек** | Просмотр возможностей на модерации, подтверждение/отклонение, просмотр деталей |
| **Управление пользователями** | Просмотр списка соискателей, работодателей, кураторов, редактирование профилей |
| **Создание кураторов** | Только для суперадмина |
| **Логика** | 🟡 Не реализована, используется статический контент из моков |

## 🔐 Переменные окружения

Создай файл `.env` в корне проекта:

```env
VITE_YANDEX_MAP_API_KEY=ваш_ключ_яндекс_карт
VITE_API_BASE_URL=http://localhost:8080/api   # URL бэкенда (для будущей интеграции)
🚀 Запуск проекта
bash
npm install
npm run dev
📝 Примечания для разработчика
Стилизация
Tailwind используется для глобальных и простых стилей

CSS Modules (.module.css) — для сложных компонентов

Цвета заданы в CSS-переменных в src/app/index.css

Импорты
Все импорты через алиас @/:

js
import { Button } from '@/components/ui/button'
import { MainPage } from '@/pages/main-page'
import { formatDate } from '@/shared/lib'
Моковые данные
Все моки лежат в src/shared/api/mock/. При подключении бэкенда нужно заменить импорт моков на вызовы API.

Трансформеры данных
Файл src/shared/api/transformers.js содержит функции для преобразования данных между форматом фронтенда (camelCase) и API (snake_case). Обязательно используйте их при работе с API!

shadcn/ui
Компоненты установлены в src/components/ui/. При добавлении новых:

bash
npx shadcn@latest add <component>
